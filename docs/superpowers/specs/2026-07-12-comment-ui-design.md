# 评论区头像修复与层次卡片设计

## 目标

修复文章页评论头像不受尺寸限制而显示过大的问题，并将评论区改为层次更清晰的卡片式界面。评论提交、登录和 Supabase 数据读取行为保持不变。

## 根因

`SupabaseComments.astro` 中的评论卡片由客户端脚本通过 `document.createElement()` 创建。Astro 的组件样式默认带有编译时作用域标记；动态创建的元素没有该标记，因而 `.comment-avatar` 等局部选择器无法匹配。全局 `img` 规则只提供 `max-width: 100%`，没有限制头像的固定尺寸。

## 方案

将动态评论节点相关的样式改为可匹配运行时节点的全局选择器，并将头像尺寸与评论元数据网格的第一列同步固定为 32px。头像使用 `object-fit: cover`，并在布局收缩时保持固定尺寸。

评论列表采用“层次卡片”样式：评论卡片保留圆角、渐变表面和轻阴影；头像、作者名与时间排列在卡片顶部；正文使用舒适行距并支持长单词与换行。评论表单沿用相同的表面层级，聚焦状态有清晰的边界提示。

## 主题与响应式

- 深色主题：深蓝黑半透明渐变表面、青色边框和适度黑色阴影，保留站点的霓虹气质。
- 浅色主题：半透明白色表面、青绿色低对比边界和轻柔的蓝灰阴影，避免使用深色主题的黑色渐变与强光晕。
- 窄屏：标题区域与表单操作区纵向排列；评论卡片与头像尺寸不变，确保信息层级稳定。

## 验证

添加一个静态回归检查，确认动态创建的头像使用稳定 class，且该 class 由全局样式设定 32px 宽高。执行格式检查、Astro 类型检查和生产构建；在深色与浅色主题下检查文章评论区的桌面与窄屏布局。

## 非目标

不修改数据库结构、评论排序、权限、登录流程或管理后台评论界面。
